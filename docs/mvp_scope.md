# MVP Scope

## 项目定位

这个项目是一个暗黑破坏神 4 Build 分析助手。它不复制暗黑核 / d2core 内容，不重新搭建 BD 数据库，而是在用户提供一个 BD 页面链接后，将该页面中的 Build 信息结构化，再基于结构化数据生成 AI 分析报告。

第一版 MVP 先验证最小闭环：

1. 手动整理一个 Build JSON。
2. 读取该 JSON。
3. 生成一份严格受 JSON 约束的 LLM Prompt。
4. 让 LLM 输出普通玩家能读懂的 Build 分析报告。

## 用户输入

第一版输入：

- 一个本地 Build JSON 文件，例如 `data/sample_build.json`。
- JSON 中可记录原始暗黑核 / d2core 页面链接 `source_url`。

未来输入：

- 用户粘贴的暗黑核 / d2core BD 页面链接。
- 可选：用户选择分析语言、分析深度、玩家阶段、是否偏新手解释。

## 系统输出

第一版输出：

- 一个完整 LLM Prompt，包含：
  - 分析规则。
  - 报告结构。
  - Build JSON。

后续输出：

- Markdown 格式 AI 分析报告。
- 可选的网页展示版本。
- 可选的结构化解析结果预览。

## 第一版不做什么

- 不做网页爬虫。
- 不使用 Playwright 解析暗黑核页面。
- 不接入真实 LLM API。
- 不做数据库。
- 不做用户系统、账号、登录。
- 不做前端页面。
- 不做 Build 强度评分的自动排名。
- 不保证示例 JSON 的游戏内容真实准确。

## 第二版建议

- 增加命令行参数，支持读取任意 Build JSON 文件。
- 接入真实 LLM API，直接生成 Markdown 报告。
- 增加 JSON schema 校验。
- 增加 `outputs/` 目录保存生成的报告。
- 设计页面解析字段映射表，为后续爬取做准备。
- 支持中文和英文报告模板。

## 第三版建议

- 使用 Playwright 解析用户提供的暗黑核 / d2core 页面。
- 增加网页界面：输入链接、查看结构化数据、生成报告。
- 增加解析失败时的人工修正界面。
- 缓存用户已分析过的 Build。
- 增加 Build 对比功能。
- 增加版本变更提醒：当赛季或版本变化时提示报告可能过期。
- 增加导出 Markdown / PDF 功能。

## 风险和边界

- 页面结构可能变化，爬虫和解析逻辑需要维护。
- 暗黑核内容有版权和使用边界，项目应只处理用户主动提供的链接和结构化摘要，不批量复制内容。
- LLM 可能幻觉，因此 Prompt 必须明确禁止编造未出现在 JSON 中的内容。
- 游戏版本变化会影响机制判断，报告应始终展示 `game_version` 和 `season`。
